# JSON Object Correlation Engine - Ideas

## Structural Fingerprinting via Hashing
Extract the "shape" of a JSON object (key names, nesting, value types) and hash it,
ignoring actual values. Two objects with the same structure get the same fingerprint
and are automatically grouped into the same type. No predefined schemas needed.

For protobufs without a spec, fingerprint on field numbers and wire types instead of
key names. Same grouping logic applies.

## Fuzzy Clustering of Structural Types
Exact fingerprinting is too strict for real data (optional fields, polymorphic values,
variable-length arrays). Layer fuzzy clustering on top: compute similarity between
fingerprints (e.g. key overlap fraction) and merge types that exceed a similarity
threshold. This lets structural variants of the same logical event type be grouped
together automatically.

## Manual Action Marking (Live Mode)
In live/watch mode, the user presses a key (e.g. 'm') to mark "I just did something."
This timestamps the action in the action log so the correlation engine can open a
candidate window and look for objects that appear shortly after. Supports free-text
labels so the user can annotate what they did.

## Baseline Import
Before performing actions, observe the object stream in a quiet period to build a
"noise profile" — the background distribution of object types and their rates. This
baseline is used to subtract expected noise from action windows, isolating objects
that are likely caused by the user's action rather than coincidence.

## Change Point Detection with `ruptures`
Use the `ruptures` library (Python) for change point detection on the object stream.
Instead of fixed time windows, detect where the statistical properties of the stream
actually shift (rate, type distribution, structural patterns). Correlate detected
change points with action timestamps to identify causal relationships. This is more
robust than fixed windows because it adapts to the data rather than relying on
guessed window sizes.
